# Notifications

## Channels

- in-app operator notifications
- email for escalations and approvals
- SMS / WhatsApp left as customer-specific integrations

## Baseline events

- DQ alerts
- equipment offline alerts
- approval required events
- work-order assignment events
- agent workflow completion / rejection

## Phase 1 policy

- all notifications must be audit logged,
- approval requests must always include a deep link to the target entity,
- autonomous actions remain disabled unless explicitly approved by policy.
